🔄
Cérémonies SCRUM
📋
Sprint Planning
⏱ 2h / début de sprint
Sélection des User Stories du Product Backlog, estimation en SP (suite de Fibonacci : 1, 2, 3, 5, 8, 13…), définition du Sprint Goal.
☀️
Daily Standup
⏱ 15 min / chaque jour
Trois questions : Qu'ai-je fait hier ? Que vais-je faire aujourd'hui ? Y a-t-il des obstacles ? Mise à jour du Sprint Burndown.
🔍
Sprint Review
⏱ 1h / fin de sprint
Démonstration des fonctionnalités développées. Validation avec les parties prenantes. Mise à jour du Product Backlog si nécessaire.
🪞
Rétrospective
⏱ 1h / fin de sprint
Ce qui a bien fonctionné, ce qui doit s'améliorer, les actions à mettre en place pour le prochain sprint (Keep / Drop / Try).
👥
Rôles SCRUM
🦅
Product Owner
Le Porteur du Projet
  • Définit et priorise le Product Backlog
  • Rédige les User Stories avec les critères d'acceptation
  • Valide les livrables à chaque Sprint Review
  • Représente les utilisateurs gabonais finaux
  • Prend les décisions fonctionnelles finales
🔧
Scrum Master
Le Facilitateur Agile
  • Anime toutes les cérémonies SCRUM
  • Supprime les obstacles (impediments)
  • Vérifie le respect du cadre SCRUM
  • Maintient à jour le Sprint Burndown
  • Protège l'équipe des interruptions extérieures
💻
Development Team
L'Équipe Technique
  • Développe les fonctionnalités acceptées
  • Estime les User Stories en Story Points
  • S'auto-organise pour livrer le Sprint Goal
  • Rédige les tests (unit + E2E)
  • Respecte la Definition of Done
✅
Definition of Done — Globale
Critères à valider pour toute User Story avant de la marquer « Done »
▶

Code & Technique

Code review approuvé par au moins 1 pair Tests unitaires écrits et passants (coverage ≥ 80%) Aucune régression détectée sur les tests E2E existants TypeScript strict — aucune erreur TS Lint ESLint passant sans warnings Migrations BDD versionnées dans /supabase/migrations RLS validée sur toutes les nouvelles tables

Fonctionnel & UX

Critères d'acceptation de la User Story vérifiés Responsive testé sur mobile (≤ 375px) et desktop Testé sur Chrome, Firefox, Safari, Samsung Internet Temps de chargement < 2s sur connexion 3G simulée Deploy Vercel Preview fonctionnel sur la PR Commentaires inline pour les logiques complexes Aucune donnée sensible exposée dans les logs client
