La référence numérique des jours fériés gabonais

Gabon Fériés est une plateforme open source qui centralise, informe et engage les citoyens gabonais autour du calendrier officiel de la République Gabonaise.

🇬🇦 Aperçu général

Au Gabon, l'information sur les jours fériés reste fragmentée : elle circule via des communiqués télévisés ou des recherches Google répétitives. Gabon Fériés comble ce vide en proposant une plateforme web ouverte, participative et ludique qui centralise les dates officielles, éduque les citoyens sur leur histoire et récompense leur engagement.

Pourquoi Supabase ?
Supabase est un BaaS (Backend as a Service) open source construit sur PostgreSQL. Il offre nativement l'authentification, la base de données, le temps réel, le stockage et les Edge Functions — tout ce dont Gabon Fériés a besoin sans gérer de serveur.

⚠️
Fêtes mobiles
Les dates des fêtes religieuses mobiles (Aïd al-Fitr, Aïd al-Adha, Lundi de Pâques…) sont mises à jour manuellement par un administrateur après l'annonce officielle ou le calcul du croissant lunaire. Elles ne sont donc confirmées qu'une fois publiées.

Cases Mystère 🔒

Les jours fériés sont présentés comme des cases verrouillées avant leur date. Le jour J, un clic débloque la fiche interactive de la date : histoire officielle, traditions gabonaises, galerie photos, et accès au quiz. Les feux d'artifice virtuels s'activent automatiquement lors des fêtes nationales (16 et 17 août, 30 août).

✍️ Soumission citoyenne

Tout utilisateur authentifié peut proposer une nouvelle date (journée chômée payée, pont officiel, etc.). Chaque soumission suit un workflow de validation avant d'être publiée sur le calendrier.

1
Remplir le formulaire
L'utilisateur indique : nom de la date, jour proposé, catégorie, description, et une URL source officielle (obligatoire — Journal Officiel, communiqué gouvernemental…).
2
En attente de vérification
La soumission passe au statut pending. L'administrateur reçoit une notification temps réel sur son tableau de bord.
3
Vérification admin
L'admin consulte la source, vérifie l'information et choisit d'approuver ou de rejeter (avec un commentaire optionnel).
4
Publication & notification
Si approuvée, la date s'affiche instantanément sur le calendrier (Supabase Realtime). L'utilisateur reçoit une notification push et email de confirmation, et débloque le badge Contributeur 🏅.

Règles anti-abus

RègleValeur
Maximum de soumissions par utilisateur / 24h5

Quiz interactifs

Chaque jour férié dispose d'un mini-quiz de 3 questions (QCM ou Vrai/Faux). Le score est calculé en temps réel et les points sont crédités automatiquement sur le profil. Un utilisateur ne peut compléter chaque quiz qu'une seule fois.

🎯
Exemple de question
« Que commémore-t-on le 30 août au Gabon ? »
A) L'Indépendance · B) La Journée de la Libération ✓ · C) La Fête du Travail · D) La Rénovation

Classement (Leaderboard)

Le classement mensuel affiche les utilisateurs les plus actifs selon leurs points cumulés. Les scores sont partageables sur WhatsApp avec un message automatique comme : « Je fais partie des 5 % de Gabonais les plus incollables sur l'histoire du pays ce mois-ci 🇬🇦 ».

⚙️ Interface d'Administration

L'interface d'administration est un CMS minimaliste permettant à une personne non-technique (RH, Responsable Communication) de mettre à jour le calendrier en moins de 2 minutes après une annonce officielle.

Gestion des Quiz

L'administrateur dispose d'une banque de questions par jour férié. Pour chaque question, il définit :

  • Le type : QCM, Vrai/Faux ou Glisser-Déposer
  • Les options de réponse (JSON) avec indication de la bonne réponse
  • Une explication affichée après la réponse (pédagogie)
  • Le nombre de points attribués (par défaut : 33 pts / question)

⚡ API & Base de données

Gabon Fériés expose une API REST et Realtime via Supabase. Elle peut être utilisée par des développeurs tiers pour intégrer le calendrier dans leurs propres applications.

Récupérer les jours fériés publiés

JavaScript · Supabase Client 
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  'https://[project-id].supabase.co',
  '[anon-key]'
)

// Récupérer tous les jours fériés publiés de l'année 2025
const { data, error } = await supabase
  .from('holidays')
  .select('id, name, date_value, category, is_paid, is_bridge')
  .eq('is_published', true)
  .eq('date_year', 2025)
  .order('date_value', { ascending: true })

console.log(data)

S'abonner aux changements en temps réel

JavaScript · Supabase Realtime 
// Écouter les nouvelles dates publiées en temps réel
const channel = supabase
  .channel('holidays-changes')
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'holidays',
      filter: 'is_published=eq.true'
    },
    (payload) => {
      console.log('Nouvelle date publiée !', payload.new)
      // Mettre à jour l'UI automatiquement
    }
  )
  .subscribe()

Sécurité — Row Level Security

Toutes les tables sont protégées par des politiques RLS PostgreSQL. Un utilisateur ne peut jamais accéder aux données d'un autre utilisateur via l'API, même en connaissant l'UUID. Les politiques sont vérifiées côté serveur, dans la base de données elle-même.

🔒
Ne jamais exposer la clé service_role
La clé anon est publique et respecte le RLS. La clé service_role bypasse le RLS — elle ne doit jamais être utilisée côté client, uniquement dans les Edge Functions côté serveur.

Edge Functions — Notifications

TypeScript · Supabase Edge Function 
// supabase/functions/send-reminder/index.ts
// Déclenchée par pg_cron chaque jour à 08h00 (Libreville)

import { serve } from 'https://deno.land/std/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js'

serve(async () => {
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  // Trouver les jours fériés de demain
  const tomorrow = new Date()
  tomorrow.setDate(tomorrow.getDate() + 1)

  const { data: holidays } = await supabase
    .from('holidays')
    .select('id, name, is_bridge')
    .eq('date_value', tomorrow.toISOString().split('T')[0])
    .eq('is_published', true)

  // Insérer les notifications en file d'attente
  for (const h of holidays ?? []) {
    await supabase.from('notifications').insert({
      type: 'reminder_d1',
      title: `🇬🇦 Demain — ${h.name}`,
      body: h.is_bridge
        ? '🌉 Pont accordé ! Profitez du long week-end.'
        : 'Jour chômé payé. Bon repos à tous ! 🎉',
      data: { holiday_id: h.id }
    })
  }

  return new Response('OK')
})
